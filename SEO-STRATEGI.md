## On-page SEO struktur (for hver blog post)

### Template per artikel

```markdown
# [H1: Indeholder primært keyword]

**Meta title** (55-60 tegn): "[Keyword first] | AI Konsulenterne"
**Meta description** (150-160 tegn): Inkluder keyword + handling + differentiering

## [H2: Sekundært keyword / spørgsmål]
## [H2: Sekundært keyword / spørgsmål]
## [H2: Sekundært keyword / spørgsmål]
### [H3: Long-tail variation]
### [H3: Long-tail variation]

**Minimum 1.500 ord** for quick-win keywords
**Minimum 2.200 ord** for pillar content
**Mindst 3 interne links** per artikel (til relevante services/cases)
**Mindst 1 ekstern link** til autoritet (McKinsey, MIT, osv.)
**FAQ-sektion** nederst (giver rich snippets)
**Struktureret data** (Article + FAQPage)
```

### Pillar + cluster struktur

```
PILLAR: "AI for danske SMV'er" (/viden-om-ai/ai-for-smv)
├── Cluster: Implementering
│   ├── Hvordan kommer vi i gang med AI
```

## Månedligt SEO-review checklist

Hver måned:
- [ ] Tjek Google Search Console for nye queries
- [ ] Check ranking på top 12 quick-win keywords
- [ ] Publicér mindst 2 blog-artikler
- [ ] Opdater én eksisterende case/side med ny information
- [ ] Tjek for 404-fejl og ryd op
- [ ] Monitor Core Web Vitals

Hver kvartal:
- [ ] Keyword-review — nye muligheder?
- [ ] Konkurrentanalyse — hvad gør Todai, Netcompany, osv.?
- [ ] Backlink-audit — hvem linker til jer?
- [ ] Content-audit — hvad performer, hvad skal opdateres?

## Backlink-strategi (ud over content)

1. **Gæsteskriv** hos dansk erhvervsmedier (Børsen, Computerworld, Version2)

3. **Podcast-appearances** (Ubidy, 24Spørgsmål, Børsen Briefing)
4. **Branche-fortegnelser** (Clutch.co DK, G2.com, Wiratech)
5. **LinkedIn content** der linker til blog-artikler
